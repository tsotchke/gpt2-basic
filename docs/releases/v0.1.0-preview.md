# GPT2-BASIC v0.1.0-preview

This is the first curated GPT2-BASIC preview release. It is a QEMU-verified DOS
release candidate, not a final real-hardware claim. The public release home is
https://github.com/tsotchke/gpt2-basic. The next release gate is physical
486-class validation; Pentium timing is useful but optional.

## What Ships

- `GPT2.EXE`, compiled inside FreeDOS from the slim production runtime.
- Six strict-quality root model directories selected by the release manifest.
- CHAT, DOSHELP, OFFICE, DEV, and PORTABLE assistant packs. CHAT, DOSHELP,
  and OFFICE ship pack-local model checkpoints; DEV and PORTABLE reuse CHAT
  weights as lightweight language packs. CHAT uses a 4096-token lexicon checkpoint trained
  for broader casual conversation. Packs now carry `HELP.TXT`, `KNOW.TXT`,
  generated `KDB.TXT`/`KDBIDX.TXT` bucketed recall files, compiled
  `KB2*.BIN` fixed-width recall pages, compact `KB2TERM.TXT` inverted term
  indexes, and editable `USER.TXT` recall notes so local text can extend
  answers without increasing model size.
- `scripts/import_assistant_notes.py` for turning plain ASCII note files into
  validated `USER.TXT` or `KNOW.TXT` rows, with optional KDB rebuild for
  bundled pack knowledge.
- `scripts/create_assistant_pack.py` for creating a complete lightweight pack
  from a folder of ASCII notes, including generated KDB/KB2 recall files and
  optional `PACKS.TXT` registration.
- Source, rebuild scripts, host verification tests, selected QEMU helpers, and
  selected evidence.
- `docs/hardware-validation.md`, which defines the non-emulator validation path.
- `hardware/HWVALID.BAT`, `hardware/HWNOTES.TXT`, generated
  `C:\GPT2\RETURN.TXT`, and host verification for future physical-machine
  captures. The returned hardware log set now includes `ASTRESS.LOG`, the
  five-pack assistant stress proof.
- A minimal `C:\GPT2` hardware-transfer bundle builder for copying the release
  payload to a physical DOS machine.
- `SHA256SUMS.txt` inside the package and a zip-level `.sha256` sidecar.

## Evidence Summary

- Automated DOS demo run: `qemu/evidence/run_main_486.log`, generated by
  `C:\GPT2.EXE --demo` inside FreeDOS.
- Default DOS fixed-point quality: PASS 10/10, average 0.969.
- Host full-suite quality for release candidates: PASS 10/10 where promoted.
- Assistant raw prompt evaluation: PASS 83/83 original prompts across CHAT,
  DOSHELP, and OFFICE, including repeat-control, local-inference, DPMI,
  release-status, troubleshooting, trust, memory, offline-limit, and emulator
  prompts. DOSHELP and OFFICE pack quality are PASS 4/4; CHAT keeps a broader
  host quality report and is gated by the raw prompt suite.
- Assistant generalist prompt evaluation: PASS 24/24 broad CHAT prompts
  covering troubleshooting, trust, long prompts, memory, release checklists,
  offline limits, emulator basics, old hardware, and small-task planning.
- Assistant pack retrieval evaluation: PASS 42/42 paraphrased retrieval
  prompts across CHAT, DOSHELP, OFFICE, DEV, and PORTABLE using local
  `KDB.TXT` and `USER.TXT` recall surfaces.
- Assistant KDB index evaluation: PASS 42/42 using the bucket fast path before
  full-KDB fallback, with candidate row scans held below 65% of full scan.
- Assistant KDB binary evaluation: PASS 42/42 using compiled `KB2*.BIN` recall
  pages before text KDB fallback, with candidate row scans held below 65% of
  full scan.
- Assistant KB2 term-index evaluation: PASS 42/42 using `KB2TERM.TXT` before
  binary bucket fallback. The term index scored 234 candidate rows out of 1617
  possible rows, a 0.145 row ratio, and touched 0.315 of full binary recall
  bytes including the term-index read.
- Assistant usefulness workflow evaluation: PASS 37/37 tasks across 9/9
  workflows, covering operator prompts, trust/offline limits, DOS setup and
  repair, hardware-transfer and emulator evidence, office handoffs, planning
  and risk, developer pack authoring, fast local recall architecture, and
  portable intelligence.
- Assistant consistency evaluation: PASS 498/498 prompt variants and 83/83
  prompt groups after wrapper and punctuation canonicalization. This catches
  regressions where harmless phrasing changes expose prompt leakage,
  repetition, token soup, or off-topic replies.
- Assistant stress gate: PASS 50/50 original prompts across CHAT, DOSHELP,
  OFFICE, DEV, and PORTABLE under QEMU 486. The CHAT coverage includes broader small-talk prompts
  for music, boredom, relaxation, friendship, food, goals, discussion topics,
  and improvement, plus eight session-memory prompts for name, goal, answer
  style, previous-turn recall, and PORTABLE recall for BASIC/C/assembly and
  high-velocity storage. The gate rejects visible prompt leakage,
  repeated chunks, token-soup output, empty answers, off-topic answers, and
  model-unavailable records. The final QEMU 486 stress run used
  `kb2_term=46`, `kb2_bucket=3`, and `none=1` recall modes, with 134 ms
  average total reply time and 80 ms average retrieval time.
- QEMU 486DX2/66 default full-resident speed: 127 tokens in 51.57 seconds, 2.46 tok/s.
- QEMU 486DX2/66 head-shortlist speed: 3.35 tok/s on the kernel gate.
- ICC readiness: ready, score 100, no actionable next actions.

## Known Limits

- Physical 486-class board timing is still pending.
- Physical assistant stress proof is pending until a real 486-class DOS
  machine returns `ASTRESS.LOG`; QEMU already rehearses the same batch path.
- Pentium hardware is not required for the solid 486-focused release; it is an
  optional scaling datapoint.
- The generic model is intentionally small; the release claims evidence-gated
  DOS inference, not modern LLM fluency.
- Historical failed candidates and repair attempts are excluded from the
  release payload, even when their evidence remains in the repository.
- Host-only assistant training corpora such as `TRAIN.TXT` and `TOKBASE.TXT`
  are excluded from runtime bundles.
- This preview is DOS-only. Windows and OS/2 assistant shells, including the
  OS/2/Warp package, are planned against the same pack format for a later
  release.

The real-DOS-session videos preserve an 80x25-style terminal view with typed
commands, prompt entry, thinking/status lines, and streamed assistant output.
`gpt2-basic-dosbox.zip` contains a ready-to-mount DOSBox tree with generated
profiles for the `ASSIST.EXE` chat shell, raw interactive completion, demo,
quality, performance, trace, and sampling runs. It includes the `CWSDPMI.EXE`
DPMI host needed by the FreeBASIC-built DOS executables. The chat shell keeps
structured session memory and exposes `/memory`, `/remember KEY=VALUE`, and
`/forget`.
`gpt2-basic-launch-kit.zip` bundles the release zips, checksums, manifest,
generated MP4s, thumbnail, release notes, public launch plan, DOSBox guidance,
and promotional copy into one deterministic handoff archive for publishing and
press work.

The `Preview Release` GitHub Actions workflow uploads the core preview zip,
hardware-transfer zip, their sidecars, and the preview manifest as the
`gpt2-basic-v0.1.0-preview` workflow artifact after rebuilding and verifying
both zips. The launch media and launch kit are generated locally for the GitHub
prerelease assets. The workflow also checks deterministic rebuilds: rebuilding
either zip without content changes must keep the same zip checksum. The preview
manifest date is pinned for reproducibility, and its package fields use
portable artifact names instead of host output paths. CI rebuilds the preview
package under a second output root and requires the same zip checksum. The
builder also refuses untracked files in copied release-input roots so local
scratch files cannot alter the zip; the hardware-transfer builder applies the
same rule to its model, pack, executable, hardware, and staged-source inputs.
Pass `--generated-date YYYY-MM-DD` only for a deliberate release respin.

The final artifact verifier is the publish gate: it checks checksums, zip
sidecars, extracted zip payloads, live tree versus zip payload consistency,
required DOS demo evidence, the exact release model/packs set, absence of
deferred media/VM payloads and transient host-cache artifacts, SHA-256 field
syntax, duplicate entries, manifest path safety, strict POSIX path
normalization, canonical checksum ordering, and deterministic ZIP entry
metadata, plus the 8.3-safe hardware transfer manifest.

After downloading all four zip/sidecar artifacts, a consumer can verify the
downloaded payloads from the extracted preview tree:

```sh
unzip gpt2-basic-preview.zip
unzip gpt2-basic-hardware-transfer.zip
cd gpt2-basic-preview
python3 scripts/verify_preview_artifacts.py \
  --preview-dir . \
  --preview-zip ../gpt2-basic-preview.zip \
  --preview-zip-sha256 ../gpt2-basic-preview.zip.sha256 \
  --hardware-dir ../gpt2-basic-hardware-transfer \
  --hardware-zip ../gpt2-basic-hardware-transfer.zip \
  --hardware-zip-sha256 ../gpt2-basic-hardware-transfer.zip.sha256 \
  --repo-manifest preview_release_manifest.md
```
