# GPT2-BASIC v0.1.0-preview

This is the first curated GPT2-BASIC preview release. It is a QEMU-verified DOS
release candidate, not a final real-hardware claim. The public release home is
https://github.com/tsotchke/gpt2-basic. The next release gate is physical
486-class validation; Pentium timing is useful but optional.

## What Ships

- `GPT2.EXE`, compiled inside FreeDOS from the slim production runtime.
- Six strict-quality root model directories selected by the release manifest.
- CHAT, DOSHELP, and OFFICE assistant packs. All three ship pack-local model
  checkpoints; CHAT uses a 4096-token lexicon checkpoint trained for broader
  casual conversation.
- Source, rebuild scripts, host verification tests, selected QEMU helpers, and
  selected evidence.
- `docs/hardware-validation.md`, which defines the non-emulator validation path.
- `hardware/HWVALID.BAT`, `hardware/HWNOTES.TXT`, generated
  `C:\GPT2\RETURN.TXT`, and host verification for future physical-machine
  captures.
- A minimal `C:\GPT2` hardware-transfer bundle builder for copying the release
  payload to a physical DOS machine.
- `SHA256SUMS.txt` inside the package and a zip-level `.sha256` sidecar.

## Evidence Summary

- Automated DOS demo run: `qemu/evidence/run_main_486.log`, generated by
  `C:\GPT2.EXE --demo` inside FreeDOS.
- Default DOS fixed-point quality: PASS 10/10, average 0.969.
- Host full-suite quality for release candidates: PASS 10/10 where promoted.
- Assistant pack quality: CHAT PASS 48/48 average 1.000; DOSHELP PASS 4/4
  average 0.954; OFFICE PASS 4/4 average 0.966. CHAT is the interactive
  default conversation pack and now emits two-sentence replies for most short
  casual prompts. It has manual QEMU evidence for casual prompts.
- QEMU 486DX2/66 default full-resident speed: 127 tokens in 51.57 seconds, 2.46 tok/s.
- QEMU 486DX2/66 head-shortlist speed: 3.35 tok/s on the kernel gate.
- ICC readiness: ready, score 100, no actionable next actions.

## Known Limits

- Physical 486-class board timing is still pending.
- Pentium hardware is not required for the solid 486-focused release; it is an
  optional scaling datapoint.
- The generic model is intentionally small; the release claims evidence-gated
  DOS inference, not modern LLM fluency.
- Historical failed candidates and repair attempts are excluded from the
  release payload, even when their evidence remains in the repository.
- This preview is DOS-only. Windows and OS/2 assistant shells, including the
  OS/2/Warp package, are planned against the same pack format for a later
  release.

- `gpt2_basic_launch_short_vertical.mp4`

The real-DOS-session videos preserve an 80x25-style terminal view with typed
commands, prompt entry, thinking/status lines, and streamed assistant output.
`gpt2-basic-launch-kit.zip` bundles the release zips, checksums, manifest,
generated MP4s, thumbnail, release notes, public launch plan, and promotional
copy into one deterministic handoff archive for publishing and press work.

The `Preview Release` GitHub Actions workflow uploads the core preview zip,
hardware-transfer zip, their sidecars, and the preview manifest as the
`gpt2-basic-v0.1.0-preview` workflow artifact after rebuilding and verifying
both zips. The launch media and launch kit are generated locally for the GitHub
prerelease assets. The workflow also checks deterministic rebuilds: rebuilding
either zip without content changes must keep the same zip checksum. The preview
manifest date is pinned for reproducibility, and its package fields use
portable artifact names instead of host output paths. CI rebuilds the preview
package under a second output root and requires the same zip checksum. The
builder also refuses untracked files in copied release-input roots so local
scratch files cannot alter the zip; the hardware-transfer builder applies the
same rule to its model, pack, executable, hardware, and staged-source inputs.
Pass `--generated-date YYYY-MM-DD` only for a deliberate release respin.

## Verification Commands

```sh
python3 -m unittest discover tests
python3 scripts/audit_exported_models.py --self-test
python3 scripts/verify_assistant_packs.py
python3 scripts/verify_hardware_capture.py --self-test
python3 scripts/stage_hardware_capture_evidence.py --self-test
python3 scripts/hardware_performance_matrix.py --self-test
python3 scripts/build_hardware_transfer.py --self-test
python3 scripts/build_launch_kit.py --self-test
bash qemu/run_hardware_capture_486.sh
python3 scripts/build_preview_release.py --self-test
python3 scripts/verify_preview_artifacts.py --self-test
python3 scripts/verify_workspace_tracking.py
python3 scripts/build_preview_release.py --force
python3 scripts/build_hardware_transfer.py --force
python3 scripts/build_promo_materials.py --force
python3 scripts/build_launch_kit.py --force
python3 scripts/verify_preview_artifacts.py
```

The final artifact verifier is the publish gate: it checks checksums, zip
sidecars, extracted zip payloads, live tree versus zip payload consistency,
required DOS demo evidence, the exact release model/packs set, absence of
deferred media/VM payloads and transient host-cache artifacts, SHA-256 field
syntax, duplicate entries, manifest path safety, strict POSIX path
normalization, canonical checksum ordering, and deterministic ZIP entry
metadata, plus the 8.3-safe hardware transfer manifest.

After downloading all four zip/sidecar artifacts, a consumer can verify the
downloaded payloads from the extracted preview tree:

```sh
unzip gpt2-basic-preview.zip
unzip gpt2-basic-hardware-transfer.zip
cd gpt2-basic-preview
python3 scripts/verify_preview_artifacts.py \
  --preview-dir . \
  --preview-zip ../gpt2-basic-preview.zip \
  --preview-zip-sha256 ../gpt2-basic-preview.zip.sha256 \
  --hardware-dir ../gpt2-basic-hardware-transfer \
  --hardware-zip ../gpt2-basic-hardware-transfer.zip \
  --hardware-zip-sha256 ../gpt2-basic-hardware-transfer.zip.sha256 \
  --repo-manifest preview_release_manifest.md
```
